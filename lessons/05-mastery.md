# Lesson 5｜熟達（Phase 5対応）

## 解説：チーム展開とコスト最適化

- CLAUDE.md / `.claude/commands/` / `.claude/skills/` はリポジトリにコミットすれば、チーム全員が同じ知識・自動化を共有できる
- コンテキストが大きくなるほどコスト・速度に影響する。大きいファイルは全読みさせず「該当箇所だけ」読ませる、要約させてから次に進む、などを意識する
- `/security-review` や `/code-review` を、PR作成前のルーティンに組み込む

## 具体例：レビューをワークフロー化する

```
1. 実装が終わったら /code-review を実行
2. 指摘を確認・修正
3. /security-review で機密情報の混入やインジェクション脆弱性がないか確認
4. 問題なければPR作成
```

## 演習

`exercises/phase5-mastery/README.md` に手順あり。

1. チームメンバーとCLAUDE.mdの内容をレビューし合い、認識のズレを解消する
2. 実際のPR作成前に `/code-review` → `/security-review` の順で回してみる
3. Claude Agent SDKのドキュメントに目を通し、Claude Code的なエージェントを自作するイメージを掴む

---

## まとめ：使い方のコツ

- **小さく確認する** — 一括承認せず、差分を都度見る
- **CLAUDE.mdを育てる** — つまずいたら書き足す。ドキュメントは"生きている"もの
- **向き不向きを見極める** — サブエージェント・Plan Mode・Hooksはそれぞれ得意な場面が違う。万能ツールとして使わない
- **公式情報を追う** — Claude Codeは更新が速いので、changelogは定期的にチェックする
