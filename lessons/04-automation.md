# Lesson 4｜自動化・並列化（Phase 4対応）

## 解説：サブエージェントと並列実行

- 独立して調査・実装できるタスクは、サブエージェント（Task/Agentツール）に分担させると速い
- 例：「フロントの調査」と「バックエンドの調査」は依存関係がないので同時に投げられる
- 逆に、**単純な1タスク**にサブエージェントを使うと、コンテキスト共有のオーバーヘッドの方が大きくなるので避ける

## 具体例：並列調査の依頼の仕方

```
このリポジトリの認証まわりについて2つ調べて：
1. フロントエンドでトークンをどう保存・送信しているか
2. バックエンドでトークンをどう検証しているか
それぞれ独立に調べられるので、並行して調査してレポートして。
```

## 具体例：git worktreeでリスクの高い変更を隔離

```bash
git worktree add ../myapp-experiment feature/risky-refactor
```

別ディレクトリで試し、問題なければmainブランチにマージ、ダメなら丸ごと破棄できる。

## 演習

`exercises/phase4-automation/README.md` に手順あり。

1. 独立した2つの調査タスクを1回のメッセージで並行依頼してみる
2. リスクのあるリファクタリングをworktreeで隔離して試す
3. `/loop` または cron的な仕組みで、定期実行タスク（例: 依存パッケージの更新チェック）を1つ組んでみる
