# Lesson 1｜基礎操作（Phase 1対応）

## 解説：6つの基本ツールを理解する

Claude Codeは会話の裏で以下のツールを組み合わせて「作業」します。何をお願いするとどのツールが動くかを知っておくと、指示の出し方が上達します。

| ツール | 役割 | こう頼むと動く例 |
|---|---|---|
| Read | ファイル内容を読む | 「`src/app.ts` の中身を見せて」 |
| Grep | 文字列・正規表現でコード検索 | 「`useEffect` を使っている箇所を全部探して」 |
| Glob | ファイル名パターンで検索 | 「`**/*.test.ts` のテストファイル一覧を出して」 |
| Write | 新規ファイル作成 | 「READMEを新規に書いて」 |
| Edit | 既存ファイルの差分編集 | 「このバグを直して」 |
| Bash | シェルコマンド実行 | 「テストを実行して結果を教えて」 |

## 具体例：CLAUDE.mdテンプレート

プロジェクト直下に置くと、以降の会話全てで自動的に読み込まれます。テンプレートは `examples/CLAUDE.md.example` にも置いてあるのでコピーして使えます。

```markdown
# CLAUDE.md

## プロジェクト概要
- Node.js + TypeScriptのWeb API。認証はJWT。

## ビルド・テスト
- インストール: `npm install`
- テスト: `npm test`
- Lint: `npm run lint`

## コーディング規約
- インデントはスペース2つ
- コミットメッセージは英語、Conventional Commits形式

## 注意点
- `legacy/` 配下は触らない（廃止予定、触ると既存テストが壊れる）
- DBマイグレーションは必ず `npm run migrate:dry` で事前確認してから適用する
```

## 演習

`exercises/phase1-basics/README.md` に手順あり。結果はそこに追記する。

1. 自分のプロジェクトに `CLAUDE.md` を作る（`/init` で雛形生成→加筆でもOK）
2. 「〇〇の実装箇所を探して」と自然言語だけでGrep/Globを使わせてみる
3. 小さなバグを1件、コードを直接見ずにClaude Codeの説明だけで理解し、修正させる
4. 生成された`git commit`メッセージを読み、規約に沿っているか確認する
