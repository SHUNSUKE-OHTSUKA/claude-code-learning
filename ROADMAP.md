# Claude Code マスターロードマップ

Claude Code（Anthropic製のAIコーディングエージェント／CLI）を初心者から使いこなせるようになるまでの学習ロードマップです。全5フェーズ＋継続的な習慣で構成しています。自分のペースで、各フェーズの「やってみる」を実際に手を動かして進めるのがコツです。

---

## Phase 0｜準備（所要目安: 半日）

**ゴール**: Claude Codeを起動し、最初の対話ができる状態にする。

- [ ] Claude Code CLI をインストール（`npm install -g @anthropic-ai/claude-code` など公式手順に従う）
- [ ] Anthropicアカウントで認証（API key or Claude subscription連携）
- [ ] 適当なリポジトリで `claude` コマンドを起動してみる
- [ ] 「このプロジェクトの構成を説明して」など簡単な質問を投げてみる
- [ ] `/help` で使えるコマンド一覧を確認する

**理解すべき概念**: エージェントとチャットボットの違い（Claude Codeはツールを実行して"作業する"）、ワーキングディレクトリの概念。

---

## Phase 1｜基礎操作（所要目安: 1週間）

**ゴール**: 日常的なコーディング作業をClaude Codeに任せられるようになる。

| トピック | 内容 |
|---|---|
| 基本ツール | Read / Write / Edit / Bash / Grep / Glob の役割を理解する |
| ファイル探索 | 「〇〇の実装箇所を探して」のような自然言語検索を使いこなす |
| 編集フロー | 小さな修正 → 差分確認 → 承認、のサイクルに慣れる |
| Git連携 | commit作成、diff確認、PR作成をClaude Codeに任せる |
| CLAUDE.md | プロジェクトルートに `CLAUDE.md` を置き、コーディング規約・ビルド手順・注意点を書く（`/init` で雛形生成） |

**やってみる**:
- [ ] バグ修正を1件、Claude Codeだけで完結させる
- [ ] `/init` でCLAUDE.mdを生成し、内容をレビュー・加筆する
- [ ] `git commit` をClaude Codeに作らせ、メッセージの質を確認する

---

## Phase 2｜効率化（所要目安: 2週間）

**ゴール**: 権限設定とコマンドを使い、対話の摩擦を減らす。

- [ ] `settings.json` / `.claude/settings.local.json` の役割を理解し、よく使うコマンドを許可リストに追加する
- [ ] **Plan Mode**（実装前に計画だけ立てさせるモード）を使い、大きめの変更で計画レビューを挟む習慣をつける
- [ ] カスタムスラッシュコマンド（`.claude/commands/*.md`）を1つ自作する（例: `/review`, `/deploy` など定型作業）
- [ ] `/code-review` や `/security-review` のような既存スキルを試す
- [ ] コンテキストの節約（不要な出力を要約させる、大きいファイルは部分読みさせる）を意識する

**理解すべき概念**: なぜ権限プロンプトが出るのか（安全性のため）、Plan Modeがなぜレビューに向くか。

---

## Phase 3｜拡張（所要目安: 3〜4週間）

**ゴール**: Claude Code自体をプロジェクトやチームに合わせて拡張できる。

| トピック | 内容 |
|---|---|
| **Skills** | `SKILL.md` で再利用可能な手順書を作る。頻出タスク（レビュー観点、デプロイ手順など）をスキル化 |
| **MCP (Model Context Protocol)** | 外部サービス（GitHub, Slack, DB, 社内API等）をMCPサーバーとして接続し、ツールとして使わせる |
| **Hooks** | 「編集後に自動でlintを走らせる」「コミット前に必ずテストする」など、決定論的な自動化をhookで実装する |
| **Memory / CLAUDE.md運用** | プロジェクト固有の知識・過去の失敗・設計判断を継続的にCLAUDE.mdへ蓄積する |

**やってみる**:
- [ ] 自分のよくあるタスクをSKILL.md化する
- [ ] 1つMCPサーバーを接続する（例: GitHub, ファイルシステム系など）
- [ ] post-edit hookでformatter/linterを自動実行させる

---

## Phase 4｜自動化・並列化（所要目安: 1〜2ヶ月）

**ゴール**: 複数のエージェントやバックグラウンド処理を組み合わせて生産性を最大化する。

- [ ] **Subagents（Task/Agentツール）**: 探索用・実装用・レビュー用などエージェントを使い分ける
- [ ] **並列実行**: 独立したタスクを複数のエージェントに同時に投げる感覚を掴む
- [ ] **Worktree分離**: 大きな変更やリスクのある変更をgit worktreeで隔離して試す
- [ ] **バックグラウンドタスク**: 長時間かかる処理（テスト実行、ビルド等）をバックグラウンドで走らせ、通知を待つ
- [ ] **スケジューリング**: `/loop` や cronベースのタスクで定期実行（監視、日次レポート等）を組む

**理解すべき概念**: いつサブエージェントを使うべきか（独立性・並列性が高いとき）、いつ使うべきでないか（単純なタスクに使うとオーバーヘッドになる）。

---

## Phase 5｜熟達（継続的）

**ゴール**: Claude Codeを"使う"だけでなく、自分やチームのワークフローに最適化された道具に育てる。

- [ ] チーム共有のCLAUDE.md / Skills / Hooksをリポジトリにコミットし、オンボーディングコストを下げる
- [ ] コスト・トークン使用量を意識した運用（モデル選択、コンテキスト管理、キャッシュ活用）
- [ ] セキュリティレビュー・コードレビューをワークフローに組み込む（`/security-review`, `/code-review`）
- [ ] Claude Agent SDKを触り、Claude Code的なエージェントを自作アプリに組み込む視点を持つ
- [ ] 公式ドキュメント・チェンジログを定期的に確認し、新機能（新ツール、新モデル）をキャッチアップする

---

## 継続的な習慣

- **小さく確認する**: 大きな変更を一括で承認せず、差分を都度レビューする
- **CLAUDE.mdを育てる**: つまずいた箇所・繰り返し説明した前提は都度追記する
- **失敗から学ぶ**: 誤った編集や無駄なツール呼び出しがあったら、指示の書き方やhook/skillで再発防止する
- **公式情報を追う**: Anthropicのドキュメント・changelogは更新頻度が高いため定期チェックが有効

---

## 学習リソース（起点）

- Anthropic公式ドキュメント（Claude Code / Claude Agent SDK）
- Claude Codeのchangelog・リリースノート

> 各フェーズの所要目安はあくまで目安です。日常的に手を動かすタスクがある人ほど早く進みます。まずはPhase 0〜1を実際に自分のリポジトリで試すところから始めてください。
